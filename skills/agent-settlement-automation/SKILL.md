# 代理对账结算自动化

## 概述

自动化处理代理渠道的月度对账结算流程，从销售明细和订单数据中提取信息，生成完整的结算表格，包括成交明细、渠道汇总、退费记录、CPS服务费计算和汇率信息。

## 业务场景

适用于需要定期对代理渠道进行结算的场景：
- 代理渠道月度对账
- CPS（按销售额付费）模式的服务费结算
- 跨境业务的汇率处理
- 退费订单的结算扣除

## 核心功能

### 1. 数据提取与筛选
- 从销售明细表筛选指定代理的成交记录
- 计算进线到成交的时间差（首签时间 - 末次渠道更新日期）
- 按时间差进行颜色标注（0-30天绿色，31-60天黄色，>60天红色）
- 从订单宽表筛选退费记录（按月份和退费类型）

### 2. 结算表生成
- **成交明细**：完整的销售数据，带时间差计算和颜色标注
- **渠道汇总**：按渠道汇总首签金额（仅统计进线-成交时间<60天的订单）
- **退费明细**：当月退费记录或"本月暂无退费"标注
- **CPS服务费表**：
  - 自动填充渠道名称、订单金额、退费金额
  - 公式计算：有效订单 = 订单金额 - 退费金额
  - CPS费用 = 有效订单 × CPS率（0.35）
  - 港币费用 = CPS费用 ÷ 汇率

### 3. 汇率自动获取
- 从国家外汇管理局网站自动抓取汇率数据
- 自动计算上月最后一个工作日（遇周末向前回溯）
- 截取汇率表格并插入到结算表
- 提取港元汇率并自动填充到CPS表K列
- 汇率格式转换：100人民币=X港元 → 1人民币=Y港元

## 技术实现

### 数据处理
- **pandas**: 数据筛选、聚合、透视表生成
- **openpyxl**: Excel文件读写、公式处理、样式设置、图片插入

### 网页自动化
- **Playwright**: 浏览器自动化（汇率查询页面操作）
- **Pillow**: 图片处理（截图插入Excel）

### 关键技术点
1. **大数据量处理**：模板sheet有百万行时，采用删除+重建策略而非逐行清空
2. **iframe操作**：汇率查询页面使用iframe，需要正确定位frame后再操作
3. **日期处理**：自动计算工作日，跳过周末
4. **公式保留**：更新数据时保留Excel公式结构
5. **颜色标注**：根据业务规则对时间差进行可视化标注

## 输入要求

### 文件结构
```
sample/
  ├── 益智海外用户销售明细_末次渠道.xlsx
  ├── 海外益智主订单宽表.xlsx
  └── 香港大v Amy2026年4月渠道结算单.xlsx (模板)
```

### 表格结构要求
- **销售明细表**：表头在第7行，包含"末次渠道名称"、"末次渠道更新日期"、"首签时间"、"首签金额"等列
- **订单宽表**：表头在第4行，包含"末次渠道名称"、"退费月份"、"学员退费类型"、"退费金额"等列
- **结算模板**：包含"渠道汇总"、"思维-Amy成交时间"、"CPS服务费"、"汇率"等sheet

## 输出结果

生成到 `output/` 目录：
- `香港大v {代理名}{年}年{月}月渠道结算单.xlsx` - 完整结算表
- `rate_{年}-{月}.png` - 汇率截图

## 使用示例

```bash
# 安装依赖
pip install pandas openpyxl playwright pillow
playwright install chromium

# 运行脚本
python process_settlement.py
```

脚本会自动处理上个月的数据并生成结算表。

## 配置说明

可在脚本中修改以下参数：
- `agent_name`: 代理名称（默认 "Amy"）
- 列索引常量：根据实际表格结构调整
- 颜色阈值：调整时间差的颜色标注规则
- CPS率：默认0.35，可根据实际合同调整

## 注意事项

1. **网络要求**：需要访问 https://www.safe.gov.cn 获取汇率数据
2. **浏览器要求**：首次运行需要安装 Playwright 的 Chromium 浏览器
3. **数据准确性**：请确保输入表格的列结构与脚本中的列索引常量匹配
4. **工作日计算**：汇率查询日期为上月最后一个工作日，周末自动向前回溯
5. **退费处理**：仅统计"未开课退费"和"试学期内退费"两种类型
