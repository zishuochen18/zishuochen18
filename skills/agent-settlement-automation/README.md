# 代理对账结算自动化

自动化处理代理渠道的月度对账结算流程，从销售明细和订单数据中提取信息，生成完整的结算表格。

## 功能特性

### Step 1: 提取代理成交数据
- 从销售明细表筛选指定代理的成交记录
- 计算"进线-成交时间"（首签时间 - 末次渠道更新日期）
- 对时间差进行颜色标注（0-30天绿色，31-60天黄色，>60天红色）

### Step 2: 筛选月度退费订单
- 从主订单宽表筛选指定代理的退费记录
- 按退费月份和退费类型过滤（未开课退费/试学期内退费）
- 提取退费金额用于结算计算

### Step 3: 生成结算表
- **3.1 成交明细**：将筛选后的销售数据写入结算表
- **3.2 渠道汇总**：生成透视表（按渠道汇总首签金额，筛选进线-成交时间<60天）
- **3.3 退费明细**：填写退费数据或标注"本月暂无退费"
- **3.4 CPS服务费表**：填充渠道名称、订单金额、退费金额，自动计算有效订单和CPS费用
- **3.5 汇率获取**：从国家外汇管理局自动抓取上月最后工作日的人民币兑港元汇率，截图并回填到K列

## 使用方法

```bash
python process_settlement.py
```

脚本会自动：
1. 读取 `sample/` 目录下的销售明细表和订单宽表
2. 处理上个月的数据（自动计算）
3. 生成结算表到 `output/` 目录

## 输入文件

需要在 `sample/` 目录准备以下文件：
- `益智海外用户销售明细_末次渠道.xlsx` - 销售明细表
- `海外益智主订单宽表.xlsx` - 主订单宽表
- `香港大v Amy2026年4月渠道结算单.xlsx` - 结算表模板

## 输出文件

生成到 `output/` 目录：
- `香港大v {代理名}{年}年{月}月渠道结算单.xlsx` - 完整结算表
- `rate_{年}-{月}.png` - 汇率截图

## 技术栈

- **pandas**: 数据筛选和聚合
- **openpyxl**: Excel 文件读写、公式处理、样式设置
- **Playwright**: 网页自动化（汇率抓取）
- **Pillow**: 图片处理（截图插入）

## 配置说明

可在脚本中修改以下常量：
- `agent_name`: 代理名称（默认 "Amy"）
- 列索引常量：根据实际表格结构调整
- 颜色阈值：调整时间差的颜色标注规则
